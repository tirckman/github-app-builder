# 功能测试指南

> 测试现有功能，确保一切正常工作

## 🚀 启动开发服务器

```bash
cd github-app-builder
npm run dev
```

访问：http://localhost:3000

## 📋 完整流程测试

### 1. 首页测试 ✅

**测试步骤**：
1. 打开 http://localhost:3000
2. 检查页面是否正常显示
3. 查看是否有渐变背景和动画效果
4. 点击"开始创建"按钮

**预期结果**：
- ✅ 页面加载流畅
- ✅ 动画效果正常
- ✅ 按钮可以点击
- ✅ 跳转到选择类型页面

**测试AI助手**：
- 点击右下角悬浮按钮（💬）
- 查看是否显示相关问题
- 测试提问功能

---

### 2. 选择类型页面测试 ✅

**测试步骤**：
1. 应该自动跳转到 `/select-type`
2. 查看6种应用类型是否显示
3. 点击任意一个类型（如"待办事项"）

**预期结果**：
- ✅ 6个类型卡片正常显示
- ✅ 悬停有动画效果
- ✅ 点击后选中状态正常
- ✅ 自动跳转到浏览模板页面

**测试AI助手**：
- 打开AI助手
- 应该显示与"选择类型"相关的问题：
  - 如何选择应用类型？
  - 不同类型的区别是什么？
  - 选择后可以修改吗？

---

### 3. 浏览模板页面测试 ✅

**测试步骤**：
1. 应该自动跳转到 `/browse-templates`
2. 查看3个模板卡片是否显示
3. 测试预览功能：
   - 鼠标悬停模板卡片
   - 点击"快速预览"按钮
   - 点击"在线演示"链接

**预期结果**：
- ✅ 3个模板正常显示
- ✅ 悬停显示"快速预览"按钮
- ✅ 预览弹窗正常打开
- ✅ 在线演示链接可点击

**测试预览功能**：
1. 点击"快速预览"按钮
2. 查看预览弹窗：
   - ✅ 是否全屏显示
   - ✅ 是否有加载动画
   - ✅ 是否可以点击全屏按钮
   - ✅ 是否可以关闭
3. 测试全屏功能：
   - 点击全屏按钮（最大化图标）
   - 查看是否全屏
   - 点击退出全屏按钮
   - 查看是否恢复正常大小

**测试AI助手**：
- 打开AI助手
- 应该显示与"浏览模板"相关的问题：
  - 如何选择模板？
  - 如何预览模板效果？
  - 评分高的模板有什么优势？

### 4. 定制化页面测试 ✅

**测试步骤**：
1. 选择一个模板后，应该跳转到 `/customize`
2. 查看页面内容：
   - 组件推荐区域
   - 颜色配置区域
   - 实时预览区域

**测试组件推荐**：
1. 查看推荐的组件列表
2. 点击复选框选择/取消组件
3. 查看选中状态是否正常

**测试颜色配置**：
1. 点击颜色选择器
2. 修改主色调、辅助色、强调色
3. 查看右侧实时预览是否更新

**测试实时预览**：
1. 修改颜色后，查看预览区域
2. 检查预览效果是否实时更新

**测试AI助手**：
- 打开AI助手
- 应该显示与"定制化"相关的问题：
  - 如何定制颜色？
  - 如何添加功能组件？
  - 如何查看实时预览？

---

### 5. 部署页面测试 ✅

**测试步骤**：
1. 点击"确认并部署"按钮
2. 应该跳转到 `/deploy`

**测试部署流程**：
1. **步骤1：授权GitHub**
   - 点击"连接GitHub账号"按钮
   - 查看是否显示连接状态
   - 点击"下一步"或自动进入下一步

2. **步骤2：创建代码仓库**
   - 输入仓库名称（如：my-awesome-blog）
   - 查看提示信息是否正确
   - 点击"创建仓库"按钮

3. **步骤3：部署到Vercel**
   - 点击"连接Vercel并部署"按钮
   - 查看说明信息

4. **步骤4：构建中**
   - 查看进度条是否正常
   - 查看步骤列表是否更新
   - 等待完成（约5秒）

5. **步骤5：部署成功**
   - 查看成功页面
   - 测试"访问网站"按钮
   - 测试"复制链接"按钮
   - 测试"创建新应用"按钮

**测试AI助手**：
- 打开AI助手
- 应该显示与"部署"相关的问题：
  - 如何部署应用？
  - 部署需要多长时间？
  - 部署失败怎么办？
